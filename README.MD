# University Management System 

Консольний додаток на мові Java, призначений для автоматизації внутрішніх процесів навчального закладу. Проєкт реалізує повний життєвий цикл управління студентами, викладачами та курсами, включаючи зарахування на дисципліни, виставлення оцінок за 12-бальною шкалою та автоматичний розрахунок середнього балу (GPA).

## Функціонал програми

Програма працює в інтерактивному режимі через консольне меню та надає такі можливості:
1. **Управління студентами та викладачами (CRUD):** Додавання нових користувачів, перегляд повної бази даних, зміна статусів.
2. **Управління курсами:** Створення навчальних дисциплін, закріплення за ними профільних викладачів.
3. **Процес навчання:** Зарахування студентів на курси (з фіксацією семестру та статусу оплати).
4. **Оцінювання та аналітика:** Виставлення оцінок за національною 12-бальною шкалою, формування табелів (транскриптів) для кожного студента та автоматичний підрахунок поточного середнього балу.
5. **Звітність:** Генерація списків боржників за оплату навчання, виведення детальної інформації про курси.


## Структура та архітектура проєкту

Код проєкту чітко розділений на пакети відповідно до архітектурних стандартів Java:

**university.entities** — базові моделі даних та сутності:
 * Person — абстрактний клас, що об'єднує спільні поля для людей (ID, ім'я, email).
 * Student — клас студента (включає статус навчання та історію курсів).
 * Teacher — клас викладача (включає посаду та заробітну плату).
 * Course — модель навчальної дисципліни.
 * Enrollment — сутність зарахування (зв'язує студента, курс, оцінку та фінансовий статус).
**university.enums** — переліки для типізації даних:
 * Grade — 12-бальна система оцінювання (від `G1` до `G12` + `NA` для незавершених курсів).
 * StudentStatus — статуси студента (`ACTIVE`, `GRADUATED`, `SUSPENDED`).
 * TeacherPosition — посади викладачів (`PROFESSOR`, `LECTURER`, `ASSISTANT`).
**university.interfaces** — інтерфейси:
 * Payable — контракт для сутностей, що пов'язані з фінансовими виплатами або розрахунками.
**university.services** — бізнес-логіка додатку:
 * Сервіси для управління сутностями (`StudentService`, `TeacherService`, `CourseService`, `EnrollmentService`).
**university.util** — допоміжні утиліти:
 * GPAUtils — логіка для математичного розрахунку середнього балу студента на основі його оцінок.
**Main.java** — точка входу в програму, яка ініціалізує тестові дані (`seedData`) та запускає циклічне консольне меню.



## Як запустити проєкт

### Вимоги
* Встановлений JDK 17 або новішої версії.

powershell
[Console]::InputEncoding = [System.Text.Encoding]::UTF8; [Console]::OutputEncoding = [System.Text.Encoding]::UTF8
